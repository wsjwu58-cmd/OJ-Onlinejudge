- 了解前后端分离开发模式，具备全栈开发能力

**项目简介：** 一套完整的在线编程判题平台，支持用户做题、代码提交、自动化判题、竞赛排名、AI 辅助判题等功能。系统采用前后端分离架构，包含用户端、管理端两套前端和统一后端服务。- 对AI技术在编程教育领域的应用有浓厚兴趣

- 学习能力强，能够快速掌握新技术

**主要技术栈：** Spring Boot 3 + MyBatis-Plus + MySQL + Redis + RocketMQ + Judge0 沙箱 + WebSocket + JWT + Vue 3 + Element Plus + Monaco Editor- 具备良好的团队协作能力和沟通能力

**核心职责与技术亮点：**

- **异步判题链路设计：** 采用 **RocketMQ 两级消息队列**实现判题全流程异步化。用户提交代码后立即返回 Pending 状态，判题任务消息投递到 `judge-task-topic`，消费者逐个测试用例调用 **Judge0 沙箱**执行判题；判题完成后将结果投递到 `database-update-topic`，由下游消费者异步落库并通过 **WebSocket** 实时推送结果到前端，实现**请求快速响应与判题计算的完全解耦**。

- **Redis + Lua 脚本保障提交并发安全：** 设计了 `submit_and_update.lua` 和 `update_result.lua` 两套 Lua 脚本，**原子性**完成提交限流（同题60秒内最多5次）、重复提交拦截（processing 锁机制）、首次 AC 标记与解题计数递增等操作，避免高并发下的数据竞态问题，相比纯数据库方案**将提交接口 QPS 承载能力提升数倍**。

- **竞赛实时排行榜（Redis ZSet）：** 使用 Redis **Sorted Set** 存储比赛排行榜（member=userId，score=得分），AC 后通过 `ZINCRBY` 原子加分，`reverseRangeWithScores` 实时获取排名，配合 Redis Key 前缀隔离比赛与普通练习的提交数据；通过 **Spring 定时任务**在比赛结束时将排行榜数据持久化到 MySQL，实现**读取 O(logN) 的实时排名**。

- **用户签到功能（Redis BitMap）：** 基于 Redis **BitMap** 实现用户每日签到与连续签到天数统计，使用 `BITFIELD` 命令获取当月签到位图并通过位运算计算连续签到天数，单用户每月仅占用 4 字节存储空间，**内存占用极低**。

- **用户登录态管理（Redis Hash + JWT）：** 管理端使用 **JWT** 令牌 + 拦截器鉴权；用户端采用 **Redis Hash** 存储用户登录信息，Token 作为 Key 映射完整用户数据，设置 30 分钟有效期实现自动过期，支持服务端主动踢人。

- **数据统计与报表导出：** 管理端实现用户增长趋势、题目新增趋势、提交记录趋势、题目通过率 Top10 等多维度统计接口，使用 **Redis 缓存**热点统计结果减少数据库压力，并基于 **Apache POI** 支持 Excel 报表导出。

### 项目二：基于 Spring AI 的 RAG 知识库与智能问答助手

**项目简介：** 集成在 OJ 系统中的 AI 智能模块，支持 AI 判题、错误分析、解题提示、自由问答等功能，并基于 RAG（检索增强生成）架构构建知识库，让 AI 回答更加精准。

**主要技术栈：** Spring Boot 3 + Spring AI + DeepSeek 大模型 + Redis Vector Store + SSE 流式响应 + PDF 文档解析

**核心职责与技术亮点：**

- **RAG 检索增强生成架构：** 基于 **Spring AI** 框架对接 DeepSeek 大模型，构建了完整的 RAG 链路——通过 `PagePdfDocumentReader` 解析 PDF 文档，使用 `TokenTextSplitter` 按 350 token 分块避免 Embedding 模型的 512 token 限制，调用 **BGE 向量模型**生成文本向量并存入 **Redis Vector Store**；用户提问时先通过 `similaritySearch` 检索 Top-K 相关知识片段，拼接为增强上下文后送入大模型生成回答，**显著提升回答的专业性和准确性**。

- **SSE 流式响应与多场景 AI 服务：** 基于 **SseEmitter** 实现服务端流式推送，用户可实时看到 AI 逐字输出。封装了 AI 判题（无测试用例时替代沙箱）、语法检查、错误分析、解题提示、自由问答共 5 种 AI 服务场景，通过线程池异步执行避免阻塞主线程。

- **对话记忆管理：** 使用 **Redis** 存储用户与 AI 的多轮对话历史（按 userId + problemId 隔离），每次请求携带历史上下文送入大模型，支持最多 10 轮对话记忆，1 小时自动过期清理，实现**有状态的多轮智能对话**。
